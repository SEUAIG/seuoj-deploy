# 数据库设计与关键数据表

## 1. 概述

| 项目 | 值 |
|------|---|
| 数据库 | MySQL 8.0 |
| 数据库名 | seuoj |
| 字符集 | utf8mb4 (unicode_ci) |
| ORM | MyBatis Plus 3.5.9 |
| 初始化脚本 | `data/init/init.sql` |
| 表数量 | 21 张 |

### 通用设计约定

- **主键**: 自增 `id` (BIGINT)
- **外部标识**: UUID `public_id` (对外暴露，避免自增 ID 泄露)
- **软删除**: `is_del` TINYINT (0=存在, 1=已删除)
- **时间戳**: `created_at` (创建时自动填充), `updated_at` (更新时自动维护)
- **唯一约束**: Generated Column `active_key`，仅对未删除记录生效
- **外键**: 定义了外键约束，ON DELETE RESTRICT

## 3. 核心表详细设计

### 3.1 用户模块

#### user_info（用户信息）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 内部主键 |
| public_id | CHAR(36) | NOT NULL | UUID，对外标识 |
| username | VARCHAR(64) | NOT NULL | 用户名 |
| email | VARCHAR(128) | NOT NULL | 邮箱 |
| password | VARCHAR(128) | NOT NULL | BCrypt 哈希密码 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

| active_email | VARCHAR(128) | GENERATED | is_del=0 时等于 email，否则 NULL |

**索引**: UNIQUE(active_username), UNIQUE(active_email), UNIQUE(public_id, is_del)

#### user_role（角色定义）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | PK |
| role_code | VARCHAR(32) | 角色编码 |
| role_name | VARCHAR(64) | 角色名称 |
| is_del | TINYINT | 软删除 |

**预置数据**: USER, ADMIN, SUPER_ADMIN, TEACHER

#### user_role_rel（用户-角色关联）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | PK |
| user_id | BIGINT | FK → user_info.id |
| role_id | BIGINT | FK → user_role.id |

## 5. 数据完整性设计

### 外键约束

所有关联表均定义了外键，使用 `ON DELETE RESTRICT` 防止误删。

### 软删除

- 所有表使用 `is_del` 字段实现软删除
- MyBatis Plus 全局配置 `logic-delete-field: is_del`
- Generated Column `active_*` 确保唯一约束仅对未删除记录生效

### Generated Column 示例

```sql
active_username VARCHAR(64) GENERATED ALWAYS AS (
    CASE WHEN is_del = 0 THEN username ELSE NULL END
) STORED
```

这允许同一 username 在删除后可以被重新注册。
