# SEUOJ 项目概览与功能介绍

## 1. 项目定位

SEUOJ（Southeast University Online Judge）是面向东南大学教学场景的在线评测系统，集成了**编程题目管理、自动评测、竞赛组织、班级教学管理**以及**AI 辅助问答**等功能，旨在为师生提供一站式的算法与编程学习平台。

系统采用多服务架构，四个子模块通过 Git Submodule 管理，使用 Docker Compose 编排部署。

## 2. 核心功能模块

### 2.1 题目库（Problem Library）

| 功能 | 说明 |
|------|------|
| 题目浏览 | 分页列表，支持按标题搜索、标签筛选 |
| 题目详情 | Markdown 渲染题面，含样例输入输出、数学公式（KaTeX） |
| 题目创建/编辑 | 管理员可创建/编辑题目元信息，上传测试数据 |
| 测试数据管理 | 上传测试点压缩包（支持 zip），配置子任务与评分规则 |
| 评测配置 | 支持 Standard / Special Judge / Interactive 三种题目类型 |
| 标签体系 | 多级标签分组：算法类型、来源、时间、特殊标记 |
| 权限控制 | 统一资源权限表（resource_permission），支持 READ/WRITE 授权 |

### 2.2 代码提交与评测（Submission & Judging）

| 功能 | 说明 |
|------|------|
| 在线代码编辑 | Monaco Editor，支持语法高亮和多语言切换 |
| 多语言支持 | C, C++11/17/20, Python 3.12, Java 17, Node.js 22, Go 1.22 |
| 异步评测 | 提交后异步判题，Semaphore 并发控制 |
| 评测结果 | Accepted / WA / TLE / MLE / RE / CE / SE / Partial 等判定 |
| 子任务评分 | 支持 min/sum 聚合策略，子任务间可定义依赖关系（DAG） |
| 提交历史 | 分页查看个人/全局提交记录，按用户/判定结果筛选 |
| 提交详情 | 独立的 submission_detail 表存储 JSON 结果和子任务评分 |

### 2.3 竞赛系统（Contest）

| 功能 | 说明 |
|------|------|
| 竞赛规则 | 支持 NOI / IOI / ACM 三种赛制 |
| 竞赛管理 | 创建竞赛、设置时间、添加题目、管理参赛者 |
| 竞赛排名 | 实时排行榜（Standings） |
| 报名/取消 | 用户自主报名与取消报名 |
| 独立提交 | 竞赛内提交与普通提交隔离（contest_submission） |

### 2.4 题单系统（Problem Set）

| 功能 | 说明 |
|------|------|
| 题单创建 | 用户可创建个人题单，包含标题、描述、公开/私有设置 |
| 题目编排 | 自定义题目顺序，支持拖拽排序 |
| 权限分享 | 通过 resource_permission 表授予其他用户 READ/WRITE 权限 |

### 2.5 班级管理（Class Management）

| 功能 | 说明 |
|------|------|
| 班级创建 | 教师/管理员创建班级 |
| 学生加入 | 学生自主加入或管理员手动添加 |
| 批量导入 | 支持 Excel 批量导入学生 |
| 作业系统 | 通过 Assignment 中间资源将题单关联到班级，支持 DRAFT/PUBLISHED/CLOSED 生命周期 |
| 公告系统 | 班级和作业级别的公告，支持附件和置顶 |
| 班级概览 | 成员数、题目完成统计、学生做题数据 |

### 2.6 用户系统（User & Auth）

| 功能 | 说明 |
|------|------|
| 注册/登录 | 邮箱注册，验证码验证（QQ 邮箱 SMTP） |
| JWT 认证 | Ed25519 签名，access token (24h) + temp token (5min) |
| 角色体系 | STUDENT / TEACHER / ADMIN / SUPER_ADMIN 四级角色 |
| 密码管理 | BCrypt 加密存储，支持修改密码和邮件重置 |
| 个人主页 | 提交热力图（Heatmap）、个人统计 |
| 用户管理 | 管理员分页查看、批量导入用户、角色设置 |

### 2.7 AI 智能问答（Agent / QA）

| 功能 | 说明 |
|------|------|
| RAG 问答 | 基于 FAISS 向量检索 + BM25 稀疏检索的混合搜索 |
| PEV Agent | Planner-Executor-Verifier 三阶段迭代式问答框架 |
| 知识图谱 | 前置知识图谱可视化与检索 |
| 智能推荐 | LLM 判断是否推荐前置知识或练习题 |
| 多轮对话 | SQLite 持久化会话，支持引用溯源（Citation） |
| 代码生成 | 根据题目描述和语言生成代码（实验性） |
| 代码执行 | 通过 judgend 服务执行代码（实验性） |
| 备课系统 | PPT/PDF 解析 → 思政融合点识别 → 内容检索 → 课件优化 |
| 多模型支持 | GPT-4o-mini / Gemini / DeepSeek / Kimi / Ollama 等 |

## 3. 用户角色与权限矩阵

系统采用两级权限模型：**角色级权限**（RBAC）+ **资源级权限**（resource_permission 表）。

| 功能 | 游客 | STUDENT | TEACHER | ADMIN | SUPER_ADMIN |
|------|:----:|:-------:|:-------:|:-----:|:-----------:|
| 浏览公开题目 | ✓ | ✓ | ✓ | ✓ | ✓ |
| 提交代码 | ✗ | ✓ | ✓ | ✓ | ✓ |
| 查看提交记录 | ✗ | ✓ | ✓ | ✓ | ✓ |
| 创建题单 | ✗ | ✓ | ✓ | ✓ | ✓ |
| 参加竞赛 | ✗ | ✓ | ✓ | ✓ | ✓ |
| 创建/管理班级 | ✗ | ✗ | ✓ | ✓ | ✓ |
| 创建/编辑题目 | ✗ | ✗ | ✓ | ✓ | ✓ |
| 上传测试数据 | ✗ | ✗ | ✓ | ✓ | ✓ |
| 用户管理 | ✗ | ✗ | ✗ | ✓ | ✓ |
| 批量导入用户 | ✗ | ✗ | ✗ | ✓ | ✓ |
| 设置 ADMIN 角色 | ✗ | ✗ | ✗ | ✗ | ✓ |

**资源级权限**：通过 `resource_permission` 表，任何拥有 WRITE 权限的用户可以对同一资源授予他人 READ 或 WRITE 权限。TEACHER 角色默认拥有所有资源的 READ 权限。

## 4. 技术栈概览

| 层级 | 技术选型 |
|------|---------|
| 前端 | React 18 + TypeScript 5.6 + Vite 5.4 + Tailwind CSS 3.4 + Redux Toolkit + TanStack React Query + shadcn/ui |
| 后端 | Spring Boot 3.2 (Java 21) + MyBatis Plus 3.5.9 + MySQL 8.0 |
| 判题端 | Rust (Axum 0.8 + Tokio) + Seccomp 沙箱 + 多语言工具链 |
| AI 问答 | Python 3.10 (FastAPI) + FAISS + BM25 + Camel-AI + 多 LLM Provider |
| 基础设施 | Docker Compose + Nginx 反向代理 |
| 认证 | JWT (Ed25519) + BCrypt |

## 5. 访问入口

| 入口 | 地址 | 说明 |
|------|------|------|
| 前端页面 | `http://localhost:2280` | Nginx 托管的 SPA |
| API 接口 | `http://localhost:2280/api/*` | 反代到 backend:8080 |
| Agent 接口 | `http://localhost:2280/agent/*` | 反代到 agentend:8002 |
| 判题端 | 内部网络 `judgend:9090` | 不对外暴露，仅 backend 调用 |
| 数据库 | 内部网络 `mysql:3306` | 不对外暴露 |
