# 管理员指南

本章面向 ADMIN 和 SUPER_ADMIN 角色。

## 用户管理

访问「更多 > 管理」（`/admin/users`），可查看系统中所有用户的列表。

列表支持分页浏览，显示用户的基本信息（用户名、昵称、邮箱、角色）。

> 此页面仅管理员和超级管理员可见。学生和教师访问会被重定向到未授权页面。

## 修改用户角色

1. 在用户管理页面找到目标用户
2. 选择要设置的角色
3. 确认修改

角色权限规则：

| 操作者 | 可设置的角色 |
|--------|-------------|
| ADMIN | STUDENT、TEACHER |
| SUPER_ADMIN | STUDENT、TEACHER、ADMIN |

注意事项：
- **SUPER_ADMIN 角色不能通过界面授予**，仅可通过数据库直接操作
- ADMIN 不能将用户设为 ADMIN（需要 SUPER_ADMIN 操作）
- 修改角色会替换用户当前的角色（一个用户同时只有一个角色）

## 批量导入用户

1. 在用户管理页面点击「批量导入」
2. 按 Excel 模板格式准备文件（参考 `data/test/batch-import/uimport-success.xlsx`）：
   - 包含用户名、邮箱等字段
   - 一卡通号等字段可选
3. 上传文件
4. 系统处理：
   - 新用户 → 创建账号
   - 已存在的用户 → 跳过
5. 查看导入结果报告（成功数、跳过数、失败数）

## 权限体系总览

```
SUPER_ADMIN
  └── ADMIN
       └── TEACHER
            └── STUDENT

资源权限（独立于角色）：
  PROBLEM / PROBLEM_SET / CLASS / CONTEST
    ├── WRITE（可编辑，蕴含 READ）
    └── READ（仅查看）
```

- **角色**决定系统级功能的访问权限（如管理页面、创建资源的能力）
- **资源权限**决定对具体资源实例的操作权限（如某个题单、某个班级）
- 管理员可以看到所有用户，但编辑具体资源仍需对应的资源权限
