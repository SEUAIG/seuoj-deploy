# 常见问题

## 提交后一直显示 Pending？

提交后系统将代码发送到判题服务器排队评测。如果长时间（超过 1 分钟）仍为 Pending，可能是：
- 判题服务器负载较高，正在处理其他提交
- 判题服务出现故障

建议刷新页面查看，如持续异常请联系管理员。

## 编译错误（CE）怎么查看？

在提交详情页（点击提交记录进入）可以看到 `error_detail` 字段，其中包含编译器的完整错误输出。

常见原因：
- 语法错误
- 选择了错误的编程语言
- 使用了目标编译器不支持的特性

## 比赛中能看到别人的提交吗？

取决于比赛设置：
- 比赛进行中，你只能看到自己的提交记录
- 比赛结束后，提交记录可能对所有参赛者可见
- 部分比赛开启了 `hide_statistics`，比赛期间排行榜不可见

## 时间限制和内存限制是多少？

每道题的时间限制和内存限制由出题者单独设置，在题目详情页可以查看。通常：
- 时间限制：1000ms ~ 3000ms
- 内存限制：256MB ~ 512MB

## 支持哪些编程语言？

| 语言 | 编译/运行方式 |
|------|--------------|
| C | gcc |
| C++ | g++ |
| Java | javac + java |
| Python | python3 |
| Go | go |
| Node.js | node |

## 怎么加入非公开班级/比赛？

非公开的班级和比赛不会出现在公开列表中。你需要：
- **班级**：由教师在班级管理中添加你为成员（或通过批量导入）
- **比赛**：由教师将比赛关联到你所在的班级，或通过资源权限授予你访问权

## 注册时收不到验证码？

- 检查邮箱地址是否正确
- 查看垃圾邮件文件夹
- 等待 1-2 分钟后重试发送
- 如持续收不到，联系管理员

## 提交代码后显示 SystemError（SE）？

SystemError 表示判题系统内部出错，不是你的代码问题。通常由以下原因导致：
- 判题服务临时故障
- 题目的测试数据配置有误

建议稍后重新提交。如果反复出现，请联系教师或管理员。

## 我的角色是学生，怎么变成教师？

角色变更需要管理员操作：
1. 联系管理员或超级管理员
2. 管理员在用户管理页面将你的角色修改为 TEACHER

你不能自行修改角色。
