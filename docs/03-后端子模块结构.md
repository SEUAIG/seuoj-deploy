## 6. 认证与安全

### JWT 实现

- **算法**: Ed25519 (EdDSA 椭圆曲线)
- **Token 类型**: ACCESS (24 小时) / TEMP (5 分钟)
- **Claims**: subject=userId, token_type, iat, exp, jti
- **密钥管理**: 环境变量注入 Base64 编码的公私钥，未配置时自动生成

### 鉴权流程

1. `JwtAuthInterceptor` 拦截所有请求，解析 JWT → 设置 `UserContextHolder` (ThreadLocal)
2. `AuthAspect` (AOP) 检查 Controller 方法上的 `@AllowAnonymous` / `@RequireRole` 注解
3. 细粒度资源权限由 `PermissionService` 在 Service 层检查
4. 角色不足时抛出 `AuthorizationException`，由 `GlobalExceptionHandler` 转为 HTTP 401/403

### 密码安全

- BCrypt 加密存储，60 字符哈希输出
- 验证码：Caffeine 缓存，300 秒过期，最多 5 次验证尝试，60 秒发送间隔

## 7. 外部集成

### 判题端客户端 (JudgeHttpClient)

```
backend → judgend (HTTP)
├── getProblemData(pid)         GET /judge/problem/{pid}
├── editProblem(request)        PATCH /judge/problem/edit
├── submitForJudging(request)   POST /judge/submission
├── uploadProblemArchive(...)   POST /judge/problem/data/{pid}
├── getProblemConfig(pid)       GET /judge/problem/config/{pid}
├── putProblemConfig(pid, dto)  PUT /judge/problem/config/{pid}
├── getProblemTree(pid)         GET /judge/problem/tree/{pid}
├── getProblemFile(pid, name)   GET /judge/problem/file/{pid}/{name}
└── deleteProblemFile(pid,name) DELETE /judge/problem/file/{pid}/{name}
```

认证方式：请求头携带 `JUDGE_SECRET`。

### 邮件服务

- SMTP: smtp.qq.com:465 (SSL)
- 用途：注册验证码、密码重置验证码

## 8. 异常处理体系

| 异常类 | HTTP 状态码 | 错误码 |
|--------|-----------|--------|
| BadRequestException | 400 | 40000 |
| CodeInvalidOrExpiredException | 400 | 40001 |
| AuthorizationException | 401 | 40100/40101 |
| ForbiddenException | 403 | 40300 |
| NotFoundException | 404 | 40400 |
| ConflictException | 409 | 40900 |
| CodeSendTooFrequentException | 429 | 42900 |
| CodeTooManyTriesException | 429 | 42901 |
| InternalServerException | 500 | 50000 |
| JudgeRemoteException | 502 | 50001 |
| CodeStorageException | 500 | 50002 |

所有异常统一由 `GlobalExceptionHandler` 捕获，返回标准 `Result<T>` 结构。
