## 4. 状态管理

### Redux Slices

| Slice | 持久化 | 核心状态 |
|-------|--------|---------|
| auth | ✓ (user, isAuthenticated, jwt) | 用户信息、登录态、JWT Token |
| code | ✓ (language, fontsize, codeFileObjectArray) | 编辑器语言、字号、代码缓存（按 pid） |
| contestCode | ✓ | 竞赛独立的代码状态 |
| problemList | ✗ | 分页参数、搜索关键词、标签筛选 |
| contestList | ✗ | 竞赛列表筛选：状态、关键词、时间、赛制 |
| submissionList | ✗ | 提交列表分页参数 |
| tags | ✗ | 标签数据 |
| verification | ✗ | 验证码 ID、发送状态、过期时间 |

### TanStack React Query

- staleTime: 60 秒
- retry: 1 次
- 全局错误处理 → Sonner Toast 提示
- 主要 Query Keys：`getTags`, `getContestDetail`, `getContestList`, `getSubmission` 等

## 5. API 服务层

### Axios 配置

- 基础 URL：环境变量 `VITE_API_BASE_URL`
- 请求拦截器：自动注入 JWT Bearer Token
- 响应拦截器：401 时重置登录态，提取错误消息

### API 模块分布

| 模块 | 路径前缀 | 主要接口 |
|------|---------|---------|
| 认证 | `/api/auth` | login, register, sendCode |
| 题目 | `/api/problem` | page, detail, create, edit, config, tree, upload |
| 提交 | `/api/submission` | submit, page |
| 竞赛 | `/api/contest` | page, detail, problems, submission, standings, register |
| 题单 | `/api/problem_set` | page, detail, create, update |
| 班级 | `/api/class` | page, detail, members, link/unlink resources |
| 用户 | `/api/user`, `/api/me` | profile, heatmap |
| Agent | `/agent` | problem/list, code_generation, rag sessions/messages |

## 10. 样式系统

- **框架**: Tailwind CSS 3.4 + `@tailwindcss/vite`
- **暗色模式**: CSS class 策略（`html.dark`），通过 CSS Variables 实现
- **配色方案**: HSL 变量，主色为 teal 系
- **组件样式**: Shadcn/Radix UI 无样式组件 + Tailwind 类名
- **工具函数**: `cn()` = `clsx` + `tailwind-merge`
- **动画**: `tailwindcss-animate` 插件

## 11. Provider 栈（渲染顺序）

```
StrictMode
  └─ BrowserRouter
      └─ HelmetProvider          (SEO / 页面标题)
          └─ QueryClientProvider  (React Query)
              └─ Provider          (Redux Store)
                  └─ PersistGate    (Redux 持久化恢复)
                      └─ Toaster    (Sonner 通知)
                          └─ App    (路由)
```
