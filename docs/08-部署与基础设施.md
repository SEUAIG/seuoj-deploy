# 部署与基础设施

## 1. 部署架构

### Docker Compose 服务编排

系统使用 `docker-compose.yml` 编排 5 个容器：

| 服务 | 镜像/构建 | 内部端口 | 外部端口 | 依赖 |
|------|----------|---------|---------|------|
| mysql | mysql:8.0 | 3306 | — | — |
| judgend | services/judgend/Dockerfile | 9090 | — | — |
| agentend | services/seuoj-qa/Dockerfile | 8002 | — | — |
| backend | services/backend/Dockerfile | 8080 | — | mysql |
| frontend | services/frontend/Dockerfile + nginx | 80 | 2280, 443 | backend, judgend, agentend |

### 启动顺序

```
mysql ─────────┐
               ├──► backend ──► frontend (nginx)
judgend ───────┘
agentend ──────────────────────► frontend (nginx)
```

backend 通过 `depends_on` 等待 mysql 启动。frontend 依赖 backend、judgend、agentend。

## 2. Nginx 反向代理配置

```nginx
server {
    listen 80;

    # 静态资源 (SPA)
    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;    # History 模式
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://backend:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
        proxy_send_timeout 60s;
        client_max_body_size 50M;
    }

    # Agent 代理
    location /agent/ {
        proxy_pass http://agentend:8002;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
        proxy_send_timeout 60s;
        client_max_body_size 50M;
    }

    # 判题端 (内部访问)
    location /internal/judgend/ {
        internal;                             # 仅 Nginx 内部可访问
        proxy_pass http://judgend:9090/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

## 5. 构建流程

### Backend (Java - 多阶段构建)

```dockerfile
# Stage 1: 构建
FROM maven:3.9.6-eclipse-temurin-21 AS builder
# 先下载依赖 (利用 Docker 缓存)
COPY pom.xml .
RUN mvn dependency:go-offline
# 再编译项目
COPY src ./src
RUN mvn clean package -DskipTests

# Stage 2: 运行
FROM eclipse-temurin:21-jre
COPY --from=builder target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

### Judgend (Rust - 多阶段构建)

```dockerfile
# Stage 1: 构建
FROM rust:latest AS builder
RUN apt-get install libseccomp-dev pkg-config
COPY . .
RUN cargo build --release

# Stage 2: 运行
FROM ubuntu:24.04
RUN apt-get install libseccomp2 gcc g++ openjdk-21-jdk python3 nodejs golang
COPY --from=builder target/release/main ./runner
ENTRYPOINT ["./runner"]
```

### Frontend (Node → Nginx)

```dockerfile
FROM node:18 AS builder
COPY package*.json .
RUN npm install
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder dist/ /usr/share/nginx/html/
COPY nginx/default.conf /etc/nginx/conf.d/default.conf
```

### Agentend (Python)

```dockerfile
FROM python:3.10-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY src/ ./src/
EXPOSE 8002
CMD ["uvicorn", "src.api_server:app", "--host", "0.0.0.0", "--port", "8002"]
```

## 6. Makefile 命令

| 命令 | 说明 |
|------|------|
| `make ensure_dirs` | 检查 .env 和 agent_config.yaml，创建 data 目录，克隆/更新 aijlib |
| `make copy_dev_assets` | ensure_dirs + 复制 judgend 开发资源到 data/judgend/ |
| `make run` | `docker compose up -d --build` |
| `make dev_run` | 复制开发资源 + `docker compose up -d --build` |
| `make clean_data` | 交互确认后删除所有数据目录 |
| `make down` | `docker compose down` |

所有目标支持可选的 `NAME` 变量用于 docker compose 项目命名。

## 7. 脚本说明

### scripts/ensure_dirs.sh

- 检查 `agent_config.yaml` 和 `.env` 是否存在，不存在则报错
- 创建目录：`data/agent`, `data/backend`, `data/mysql`, `data/judgend`
- 从 GitHub 克隆 `aijlib` 到 `data/judgend/aijlib`（已存在则 git pull）

### scripts/copy_dev_assets.sh

- 将 `services/judgend/assets/*` 复制到 `data/judgend/`
- 用于开发环境初始化示例题目数据

### scripts/clean_data.sh

- 交互确认后 `sudo rm -rf data/agent data/backend data/judgend data/mysql`
- 用于完全重置数据

## 8. MySQL 配置

### my.cnf

```ini
[client]
default-character-set = utf8mb4

[mysql]
default-character-set = utf8mb4

[mysqld]
character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci
character-set-client-handshake = FALSE
```

### 初始化

- 挂载 `data/init/init.sql` 和 `data/init/seed.sql` 到 `/docker-entrypoint-initdb.d/`
- MySQL 首次启动时按字母顺序执行，先建表 (init.sql)，再插入种子数据 (seed.sql)

## 10. 网络与安全

### 端口暴露

| 端口 | 服务 | 对外 | 说明 |
|------|------|------|------|
| 2280 | frontend/nginx | ✓ | HTTP 入口 |
| 443 | frontend/nginx | ✓ | HTTPS 入口（预留） |
| 3306 | mysql | ✗ | 数据库内部 |
| 8080 | backend | ✗ | API 内部 |
| 9090 | judgend | ✗ | 评测内部 |
| 8002 | agentend | ✗ | Agent 内部 |

### 安全措施

- 仅 Nginx 暴露端口，其余服务通过 Docker 内部网络通信
- 判题端使用 Seccomp 沙箱 + Docker 容器双重隔离
- JWT Ed25519 签名认证
- BCrypt 密码加密
- 服务间通信使用 Secret Key 认证
- Nginx `internal` 指令限制判题端直接访问

### 安全待改进

| 风险 | 说明 | 建议 |
|------|------|------|
| 无 HTTPS | 443 端口未配置证书 | 配置 Let's Encrypt |
| 无限流 | API 无 rate limiting | 添加 Nginx limit_req |
| Agent 无鉴权 | agentend 直接接受 user_id | 校验 JWT |
| 固定验证码 | 开发模式可绕过邮件验证 | 生产环境关闭 |
| Secret 明文 | .env 中的密钥为明文 | 使用 Docker Secrets / Vault |

## 11. 监控与日志

### 当前状态

| 组件 | 日志位置 | 格式 |
|------|---------|------|
| backend | data/backend/log/ | Spring Boot 日志 (10MB 滚动, 7 天保留) |
| judgend | data/judgend/logs/ | tracing 每日滚动 |
| agentend | Docker stdout | loguru |
| mysql | Docker stdout | MySQL 日志 |
| nginx | Docker stdout | access/error log |

### 缺失项

- 无集中式日志收集（ELK/Loki）
- 无指标监控（Prometheus/Grafana）
- 无告警系统
- 无健康检查探针（仅 judgend 有 /health）
- 无 APM 链路追踪
