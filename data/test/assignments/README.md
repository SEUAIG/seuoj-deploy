# 作业测试

## 前置条件

- dev 环境运行中
- 预置班级1 下 4 个作业（PUBLISHED×2、DRAFT×1、CLOSED×1）
- student1~student6 是班级1的成员
- teacher1 拥有班级1的 WRITE 权限

## 测试用例

### AS1 — 查看作业列表（学生视角）

- **角色**：student1（班级1成员）
- **步骤**：
  1. 登录 student1
  2. 进入班级1详情
  3. 查看作业列表
- **预期**：
  - 能看到 PUBLISHED 作业：
    - 作业1「第一周练习：基础算法」（截止 2026-04-30）
    - 作业2「第二周练习：进阶算法」（无截止时间）
  - 能看到 CLOSED 作业：
    - 作业4「寒假预习作业」（标记为已关闭）
  - **不能看到** DRAFT 作业3「第三周练习：图论入门」

### AS2 — 查看作业详情

- **角色**：student1
- **步骤**：
  1. 点击作业1「第一周练习：基础算法」
- **预期**：
  - 显示作业描述和 Markdown 介绍
  - 截止时间显示为 2026-04-30 23:59:59
  - 题目列表显示 3 道题：
    1. P1001「区间求和」（权重 1）
    2. P1003「括号序列的深度」（权重 1）
    3. P1004「两数之和」（权重 2）

### AS3 — 作业中提交代码

- **角色**：student1
- **步骤**：
  1. 在作业1 中点击 P1001「区间求和」
  2. 编写代码并提交
- **预期**：
  - 提交成功
  - 提交记录的 `assignment_id` 关联到作业1
  - 返回作业详情可看到提交状态更新

### AS4 — 已关闭作业不可提交

- **角色**：student1
- **步骤**：
  1. 进入作业4「寒假预习作业」（状态 CLOSED）
  2. 尝试提交代码
- **预期**：
  - 不允许提交（提交按钮禁用或提交被拒绝）
  - 可以查看题目和历史提交

### AS5 — 创建作业

- **角色**：teacher1
- **步骤**：
  1. 登录 teacher1
  2. 进入班级1 的作业管理
  3. 创建新作业：标题「测试作业」，描述「测试用」
  4. 设置截止时间为未来某天
- **预期**：
  - 作业创建成功
  - 默认状态为 DRAFT
  - 学生暂时看不到

### AS6 — 为作业添加题目

- **角色**：teacher1
- **步骤**：
  1. 进入刚创建的作业编辑页
  2. 添加题目 P0001（排序1，权重1）和 P0002（排序2，权重2）
  3. 保存
- **预期**：
  - 题目列表更新
  - 排序和权重正确显示

### AS7 — 发布作业

- **角色**：teacher1
- **步骤**：
  1. 将 DRAFT 状态的「测试作业」发布
- **预期**：
  - 状态变为 PUBLISHED
  - 班级成员（student1 等）刷新后可以在作业列表中看到

### AS8 — 关闭作业

- **角色**：teacher1
- **步骤**：
  1. 将 PUBLISHED 状态的作业2「第二周练习：进阶算法」关闭
- **预期**：
  - 状态变为 CLOSED
  - 学生不能再提交该作业的题目

### AS9 — 作业总览（教师统计）

- **角色**：teacher1
- **步骤**：
  1. 进入班级1 的作业1 总览
- **预期**：
  - 显示每个学生的完成情况：
    - student1：P1001 AC (100分)、P1003 WA (0分)
    - student2：P1001 AC (100分)
    - 其他学生：无提交
  - 显示班级整体的通过率

### AS10 — 作业公告

- **角色**：teacher1
- **步骤**：
  1. 进入作业1「第一周练习：基础算法」
  2. 发布公告：标题「提交提示」，内容「注意边界条件」
- **预期**：
  - 公告创建成功
  - 预置公告「第一周作业补充说明」仍在
  - 学生在作业详情中可见新公告

### AS11 — 批量导入题目到作业

- **角色**：teacher1
- **步骤**：
  1. 进入作业编辑页
  2. 使用批量导入功能添加题目（如有导入入口）
- **预期**：
  - 题目批量添加成功
  - 排序和权重正确
