# 班级测试

## 前置条件

- dev 环境运行中
- 预置 3 个班级、成员关系、公告
- teacher1 拥有班级1和班级2的 WRITE 权限
- teacher2 拥有班级3的 WRITE 权限

## 测试用例

### CL1 — 浏览班级列表

- **角色**：student1
- **步骤**：
  1. 登录 student1
  2. 访问 `/class`
- **预期**：
  - 能看到公开班级（班级1「数据结构与算法 2026春」、班级3「C语言程序设计 2026春」）
  - 能看到自己已加入的班级（包括非公开的班级2「算法竞赛训练营」，student1 是成员）
  - 列表显示班级名称和描述

### CL2 — 查看班级详情

- **角色**：student1（班级1成员）
- **步骤**：
  1. 点击班级1「数据结构与算法 2026春」
- **预期**：
  - 显示 Markdown 介绍（课程简介、考核方式等）
  - 显示公告列表（置顶公告「欢迎加入」在上方）
  - 显示作业列表（至少 2 个 PUBLISHED 作业）
  - 显示关联的比赛

### CL3 — 查看公告

- **角色**：student1
- **步骤**：
  1. 在班级1详情中查看公告
- **预期**：
  - 公告1「欢迎加入数据结构与算法课程」置顶显示
  - 公告2「第二周课程延期通知」正常显示
  - 公告内容为 Markdown 格式渲染

### CL4 — 加入公开班级

- **角色**：student5（不在班级3中）
- **步骤**：
  1. 登录 student5（吴同学）
  2. 访问班级3「C语言程序设计 2026春」
  3. 点击「加入班级」
- **预期**：
  - 加入成功
  - 刷新后在成员列表中可以看到 student5

> 注意：根据 seed 数据，student5 已在班级1中，但不在班级3中

### CL5 — 创建班级

- **角色**：teacher1
- **步骤**：
  1. 登录 teacher1
  2. 访问 `/class`，点击「创建班级」
  3. 填写：名称「测试班级」，描述「用于测试」，公开
  4. 提交
- **预期**：
  - 班级创建成功
  - 在班级列表中出现
  - teacher1 自动获得 WRITE 权限

### CL6 — 编辑班级

- **角色**：teacher1（班级1的 WRITE 权限）
- **步骤**：
  1. 进入班级1详情
  2. 点击「编辑」
  3. 修改描述为「已更新的描述」
  4. 保存
- **预期**：
  - 描述更新成功
  - 刷新后显示新描述

### CL7 — 批量导入学生（正常）

- **角色**：teacher1
- **步骤**：
  1. 进入班级1或新创建的班级
  2. 点击「批量导入」
  3. 上传 `data/test/batch-import/simport-success.xlsx`
- **预期**：
  - 导入成功
  - 报告显示创建/加入的学生数量
  - 成员列表中出现新成员

### CL8 — 批量导入学生（重复）

- **角色**：teacher1
- **步骤**：
  1. 在刚导入过的班级中再次点击「批量导入」
  2. 上传 `data/test/batch-import/simport-reuse.xlsx`
- **预期**：
  - 已存在的成员被跳过
  - 报告显示跳过的数量

### CL9 — 移除学生

- **角色**：teacher1
- **步骤**：
  1. 进入班级1的成员列表
  2. 移除 student6（郑同学）
- **预期**：
  - 移除成功
  - 成员列表不再显示 student6
  - student6 登录后看不到该班级的非公开内容

### CL10 — 关联比赛

- **角色**：teacher1
- **步骤**：
  1. 进入班级1详情
  2. 关联比赛2「算法周赛 #5」
- **预期**：
  - 关联成功
  - 班级详情中显示比赛入口
  - 班级成员可通过班级进入比赛

### CL11 — 关联题单

- **角色**：teacher1
- **步骤**：
  1. 进入班级1详情
  2. 关联题单1「新手入门题单」
- **预期**：
  - 关联成功
  - 班级详情中显示题单入口

### CL12 — 发布公告

- **角色**：teacher1
- **步骤**：
  1. 进入班级1详情
  2. 点击「发布公告」
  3. 填写标题「测试公告」，内容「这是一条测试公告」
  4. 设为非置顶
  5. 发布
- **预期**：
  - 公告出现在列表中
  - 排在置顶公告之后
  - 班级成员刷新后可见

### CL13 — 班级总览

- **角色**：teacher1
- **步骤**：
  1. 进入班级1详情的总览视图
- **预期**：
  - 显示各学生的作业完成情况
  - student1 有提交记录（作业1 的 P1001 AC，P1003 WA）
  - student2 有提交记录（作业1 的 P1001 AC）

### CL14 — 非成员访问非公开班级

- **角色**：student5（不在班级2中）
- **步骤**：
  1. 登录 student5
  2. 尝试访问班级2「算法竞赛训练营」（非公开）
- **预期**：
  - 不可见或提示无权限

> 注意：根据 seed 数据 student5 在班级1中但不在班级2中
