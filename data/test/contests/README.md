# 比赛测试

## 前置条件

- dev 环境运行中
- 预置 3 场比赛（已结束/进行中/未开始）
- student1~student4 已报名比赛2，student1~student6 已报名比赛1

## 测试用例

### C1 — 浏览比赛列表（游客）

- **步骤**：
  1. 不登录，访问 `/contest`
- **预期**：
  - 列表中显示公开比赛（比赛1、比赛2）
  - 非公开比赛（比赛3）不显示
  - 各比赛显示标题、赛制、时间、状态

### C2 — 查看已结束比赛

- **角色**：student1（已报名比赛1）
- **步骤**：
  1. 登录 student1
  2. 进入比赛1「2026 春季训练赛」
- **预期**：
  - 可查看题目列表（P0001、P0002、P1001）
  - 可查看排行榜
  - 可查看历史提交记录
  - 排行榜中 student1 排名靠前（AC 了 P0001 和 P0002）

### C3 — 报名进行中比赛

- **角色**：student5（未报名比赛2）
- **步骤**：
  1. 登录 student5
  2. 进入比赛2「算法周赛 #5」
  3. 点击「报名」
- **预期**：
  - 报名成功
  - 可以看到比赛题目列表

### C4 — 比赛中提交代码

- **角色**：student1（已报名比赛2）
- **步骤**：
  1. 登录 student1
  2. 进入比赛2
  3. 选择第一道题（P1002「最大子段和」）
  4. 编写代码并提交
- **预期**：
  - 提交成功
  - 提交出现在比赛提交列表中
  - 提交关联到比赛（contest_submission 表有记录）

### C5 — 查看排行榜（已结束比赛）

- **角色**：student1
- **步骤**：
  1. 进入比赛1
  2. 点击排行榜
- **预期**：
  - ACM 赛制：显示各参赛者的通过题数、罚时
  - student1 通过了 A(P0001) 和 B(P0002)
  - student2(赵同学) 通过了 A(P0001)，B(P0002) TLE
  - student3(孙同学) 通过了 A(P0001)，C(P1001) RE

### C6 — 创建比赛

- **角色**：teacher1
- **步骤**：
  1. 登录 teacher1
  2. 访问 `/contest/create`
  3. 填写：标题「单元测试赛」，赛制 ACM，公开，开始/结束时间
  4. 创建
- **预期**：
  - 比赛创建成功
  - 自动跳转到编辑页面
  - 在比赛列表中可见

### C7 — 编辑比赛（添加题目）

- **角色**：teacher1
- **步骤**：
  1. 进入刚创建的比赛编辑页面
  2. 添加题目 P0001、P0002
  3. 保存
- **预期**：
  - 题目列表更新为 2 道题
  - 比赛详情页显示对应题目

### C8 — 非公开比赛访问

- **角色**：student1
- **步骤**：
  1. 登录 student1
  2. 尝试访问比赛3（`/contest/3`），该比赛非公开
- **预期**：
  - 如果 student1 没有权限，则不能查看或提示无权限
  - 如果通过班级关联有权限，则可以查看

### C9 — 取消报名

- **角色**：student1（已报名比赛2）
- **步骤**：
  1. 登录 student1
  2. 进入比赛2
  3. 点击「取消报名」
- **预期**：
  - 取消成功
  - 不再出现在参赛者列表中

### C10 — 比赛提交记录

- **角色**：student1
- **步骤**：
  1. 登录 student1
  2. 进入比赛1 的提交记录列表
- **预期**：
  - 能看到自己的 3 条提交（P0001 AC、P0002 WA、P0002 AC）
  - 可以点击查看提交详情

### C11 — IOI 赛制部分分

- **角色**：student1（比赛2 为 IOI 赛制）
- **步骤**：
  1. 在比赛2 中提交一道题，部分测试点通过
- **预期**：
  - 得分为 0~100 之间的部分分
  - 排行榜中显示该题的最高分（多次提交取最高）
