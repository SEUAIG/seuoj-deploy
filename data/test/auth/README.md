# 认证流程测试

## 前置条件

- dev 环境运行中（`http://localhost:2281`）
- 邮箱验证码开发模式已开启

## 测试用例

### A1 — 正常登录

- **步骤**：
  1. 访问 `/login`
  2. 输入用户名 `admin`，密码 `password123`
  3. 点击「登录」
- **预期**：
  - 登录成功，跳转到首页 `/home`
  - 右上角显示头像和昵称「超级管理员」
  - 导航栏「更多」下拉菜单中出现「管理」入口

### A2 — 错误密码

- **步骤**：
  1. 访问 `/login`
  2. 输入用户名 `admin`，密码 `wrongpassword`
  3. 点击「登录」
- **预期**：
  - 登录失败
  - 页面显示错误提示（密码错误）
  - 停留在登录页面

### A3 — 注册新用户

- **步骤**：
  1. 访问 `/signup`
  2. 填写用户名 `testuser1`，邮箱 `testuser1@test.local`，密码 `Test123456`
  3. 点击「发送验证码」
  4. 输入验证码（dev 模式下使用固定验证码）
  5. 点击「注册」
- **预期**：
  - 注册成功
  - 自动登录，跳转到首页
  - 默认角色为 STUDENT

### A4 — 重复邮箱注册

- **步骤**：
  1. 访问 `/signup`
  2. 填写用户名 `duplicate1`，邮箱 `admin@seuoj.local`（已被 admin 使用），密码 `Test123456`
  3. 尝试注册
- **预期**：
  - 注册失败
  - 提示邮箱已被注册

### A5 — 忘记密码重置

- **步骤**：
  1. 访问 `/forget`
  2. 输入邮箱 `student1@seuoj.local`
  3. 点击「发送验证码」
  4. 输入验证码（dev 模式固定码）
  5. 输入新密码 `NewPassword123`
  6. 点击「重置密码」
- **预期**：
  - 密码重置成功
  - 用新密码 `NewPassword123` 可以登录
  - 旧密码 `password123` 不能登录

> 注意：此测试会修改 student1 的密码，后续测试如需使用 student1 请先重置环境

### A6 — 修改密码

- **步骤**：
  1. 用 `student2` / `password123` 登录
  2. 进入修改密码功能
  3. 输入当前密码 `password123`
  4. 输入新密码 `Changed123`
  5. 确认修改
- **预期**：
  - 密码修改成功
  - 需要用新密码重新登录

### A7 — 未登录访问受保护页面

- **步骤**：
  1. 确保未登录（清除登录状态）
  2. 直接访问 `/personal`
- **预期**：
  - 被重定向到 `/login`
  - 同理访问 `/submission`、`/problemsLibrary/create` 等页面也应重定向

### A8 — 不同角色登录验证

- **步骤**：
  1. 分别用以下账号登录并检查界面差异：
     - `student1`（STUDENT）
     - `teacher1`（TEACHER）
     - `manager`（ADMIN）
     - `admin`（SUPER_ADMIN）
- **预期**：
  - 所有角色均能登录成功
  - ADMIN/SUPER_ADMIN 可见「更多 > 管理」入口
  - STUDENT/TEACHER 不可见「更多 > 管理」入口
